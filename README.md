# B站评论上下文补全(Bilibili)

这是一个面向 B 站评论区的油猴脚本，用来补全楼中楼子回复的最小上下文。

当某条子回复实际是在回复另一条子回复时，鼠标悬停在这条回复上，脚本会在附近弹出一个小浮窗，展示这条回复的最小链路：

- 主评论
- 被回复的那条子回复
- 当前回复

脚本的目标不是重建整棵评论树，而是尽量用最少的信息，快速回答“这条回复到底在回谁”。

## 主要特性

- 悬停触发，无需额外按钮
- 只对“回复子回复”的评论显示上下文
- 支持主评论、父回复、当前回复的最小链路展示
- 支持主评论和父回复的本地查找、缓存命中与接口分页补全
- 对接口空结果、父回复缺失、主评论缺失做降级处理
- 支持表情渲染与浮窗树状缩进展示

## 适用场景

适用于 B 站视频页、列表页和番剧播放页评论区，尤其适合楼中楼较深、回复关系不直观的评论串。

## 当前交互

- 鼠标悬停在目标子回复上弹出浮窗
- 主评论默认折叠，支持在浮窗内展开
- 浮窗展示的是最小上下文链，不展示兄弟回复和完整树

## 安装方式

1. 安装支持用户脚本的浏览器扩展，例如 Tampermonkey
2. 新建脚本并粘贴 `Bilibili Comment Context Completion.js` 内容
3. 打开 B 站评论区使用

## 说明

为了兼容 B 站不同评论组件结构，脚本内部保留了多层兜底逻辑，包括缓存、线程内查找、可见区扫描和必要时的接口分页查询。
